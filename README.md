### 🔍 Программирование

#### [Призрачный аргумент: как адрес функции пробрался в третий параметр](https://github.com/cr0k0Hub/Papers/blob/1e955431cdc46bfdbb36e7e8ed54f91c101f85ea/C/ghost_pointer/%D0%9F%D1%80%D0%B8%D0%B7%D1%80%D0%B0%D1%87%D0%BD%D1%8B%D0%B9%20%D0%B0%D1%80%D0%B3%D1%83%D0%BC%D0%B5%D0%BD%D1%82%3A%20%D0%BA%D0%B0%D0%BA%20%D0%B0%D0%B4%D1%80%D0%B5%D1%81%20%D1%84%D1%83%D0%BD%D0%BA%D1%86%D0%B8%D0%B8%20%D0%BF%D1%80%D0%BE%D0%B1%D1%80%D0%B0%D0%BB%D1%81%D1%8F%20%D0%B2%20%D1%82%D1%80%D0%B5%D1%82%D0%B8%D0%B9%20%D0%BF%D0%B0%D1%80%D0%B0%D0%BC%D0%B5%D1%82%D1%80.md)
*Эксперимент с указателями на функции в C, неопределённое поведение и соглашения о вызовах System V AMD64 ABI.*

В этой статье я на живом примере показываю, что происходит, если вызвать функцию с меньшим количеством аргументов, чем она ожидает, используя указатель с непрототипным объявлением. С помощью отладчика и ассемблера мы находим, откуда берётся «призрачный» третий аргумент, и разбираемся, почему компилятор пропускает такую ошибку.

**Ключевые темы:**
- указатели на функции;
- неопределённое поведение (UB);
- System V AMD64 ABI;
- передача аргументов через регистры;
- анализ ассемблерного кода.
