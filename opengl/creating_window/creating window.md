# Создание окна в OpenGL

- Автор: [cr0k0Hub](https://github.com/cr0k0Hub)
- Дата: 14.04.2026

Прежде всего необходимо создать контекст OpenGL. В этом нам поможет GLFW.

> GLFW - кроссплатформенная апишка, которая позволяет нам создавать окна и управлять контекстом

Чтобы воспользоваться GLFW необходимо перейти по ссылке https://www.glfw.org/download.html и банально его скачать. Если ты гордый пользователь шиндовс и пользуешься Visual Stuido - всё отлично. Можно смело скачать заранее скомпилированный бинарник, который можно будет вставить в свой проект.

Для таких линуксовых ребяток есть другой путь:
``` bash
sudo apt install libglfw3-dev
```
Ну или же Вы имеете абсолютное право использовать любую другую систему управления пакетами. Речь сейчас не об этом

Далее в своём проекте добавим хэдер glfw.
``` c++
#include <GLFW\glfw3.h>
```

Для корректной компиляции через GCC необходимо указать добавить следующее:
``` bash
-lglfw3 -lGL -lX11 -lpthread -lXrandr -lXi -ldl
```

Но это даже близко не всё. Есть определённая проблема - OpenGL является лишь стандартом. Конкретные его реализации могут отличаться друг от друга и это ответственность уже производителя драйвера видеокарты. Получается, что те или иные функции OpenGL, которые нам нужны, на момент компиляции неизвестно где лежат, и для решения этой задачи нам пригодится GLAD. Скачать его можно [тут](https://glad.dav1d.de/)

Вот небольшой [туториал](https://rpxomi.github.io/) по подключению GLAD: какие настройки выбрать, куда положить содержимое архива. Но тем не менее, на всякий случай опишу весь этот путь. 

После перехода по ссылке и выбора настроек мы получим архив. Архив будет содержать в себе две директории: include/ и src/, которые нам нужно будет положить в наш проект, но обо всём по порядку.
```
Выбираем следующие настройки:
Language - C/C++
Specification - OpenGL
API:
    gl - Version 3.3
    остальное пропускаем
Profile - Core
```

Остальное можно смело оставить так, как есть. Переходим в самый конец страницы и нажимаем Generate. Здесь можно будет скачать архив и распаковать. Но куда? Предположим, что у нас есть проект create_window. Его структура следующая:
```
create_window/
    main.cpp
```

Т.е. наш проект буквально сейчас состоит только из плюсового файла (предположим, что всё является именно таковым). Сразу же создадим директорию, которую назовём src и перенесём туда наш файл main.cpp (или как он там будет у Вас называться). Далее создадим директорию lib, а в ней создадим директорию glad. Теперь наш проект выглядит так:
```
craete_window/
    lib/
        glad/
    src/
        main.cpp
```

Теперь как раз в наш create_window/lib/glad/ и нужно добавить содержимое архива, а именно: include и src. Теперь наш проект выглядит так:
```
craete_window/
    lib/
        glad/
            include/
            src/
    src/
        main.cpp
```

Перейдём в наш основной файл (main.cpp) и добавим туда ещё одну строчку:
``` c++
#include <glad/glad.h>
```
Также, если Вы используете CMake, то добавим всё необходимое в CMakeLists.txt
``` c++
cmake_minimum_required(VERSION 4.2)
project(create_window)

set(CMAKE_CXX_STANDARD 23)

add_executable(create_window src/main.cpp
        lib/glad/include/glad/glad.h)

# --- Подключаем GLFW ---
find_package(glfw3 REQUIRED)

# --- Подключаем GLAD ---
# Добавляем заголовочные файлы GLAD
include_directories(${CMAKE_SOURCE_DIR}/lib/glad/include)
# Собираем исходный код GLAD в отдельную библиотеку
add_library(glad STATIC ${CMAKE_SOURCE_DIR}/lib/glad/src/glad.c)

# --- Линкуем библиотеки ---
target_link_libraries(create_window glfw glad ${GLFW_LIBRARIES})
```

Великолепно! Теперь попробуем скомпилировать наш проект и запустить. Если никаких ошибок не возникло, то идём дальше!

ВАЖНО! Заголовочный файл glad.h должен добавляться ПЕРЕД glfw.h!!!

Теперь же нам нужно в функции main() создать экземпляр окна. Делается это предельно просто:
``` c++
int main() {
    // инициализируем glfw
    glfwInit();
    // указываем, что мы будем использовать OpenGL 3.x версии
    glfwWindowHint(GLFW_CONTEXT_VERSION_MAJOR, 3);
    // указываем, что мы будем использовать OpenGL x.3 версии. В данном случае, получается, что мы используем OpenGL 3.3
    glfwWindowHint(GLFW_CONTEXT_VERSION_MINOR, 3);
    // указывем, что используем Core
    glfwWindowHint(GLFW_OPENGL_PROFILE, GLFW_OPENGL_CORE_PROFILE);

    return 0;
}
```

Далее необходимо создать объект окна, который будет хранить в себе все данные, связанные с окном. Также этот объект мы будем передавать в некоторые функции
``` c++
// создаём объект окна размером 800x600 пикселей, с названием CREATING WINDOW OR SMTH. Всё остальное будет NULL
GLFWwindow* window = glfwCreateWindow(800, 600, "CREATING WINDOW OR SMTH", nullptr, nullptr);
// если окно не создалось, то выводим сообщение об этом и очищаем всё, что связано с glfw
if (window == nullptr) {
    std::cout << "[-] failed to create GLFW window" << std::endl;
    glfwTerminate();
    return -1;
}
// создать контекст нашего окна
glfwMakeContextCurrent(window);
```

Если всё пройдёт успешно, то мы будем иметь объект окна, который будет храниться под названием window. 

Дальше нам нужно инициализировать GLAD. 
``` c++
// если не получилось найти glfwGetProcAddress - вывести ошибку
if (!gladLoadGLLoader(reinterpret_cast<GLADloadproc>(glfwGetProcAddress))) {
    std::cout << "[-] failed to init GLAD" << std::endl;
    return -1;
}
```

Теперь укажем размеры окна для OpenGL, чтобы он знал с чем работает и было где рисовать.
``` c++
// говорим OpenGL размеры окна
glViewport(0, 0, 800, 600);
```

Нам осталось совсем немного. Чтобы была возможность менять размер окна - необходимо зарегистрировать коллбек. Этим и займёмся. Объявим прототип функции в самом начале, до функции main
``` c++
// прототип коллбека для изменения размера окна
void framebuffer_size_callback(GLFWwindow* window, int width, int height);
```

А уже ниже, после функции main, напишем реализацию коллбека
``` c++
// реализация коллбека
void framebuffer_size_callback(GLFWwindow* window, int width, int height) {
    glViewport(0, 0, width, height);
}
```

В main скажем GLFW, что для изменения размера окна мы будем использовать именно наш коллбек
``` c++
// указываем наш коллбек
glfwSetFramebufferSizeCallback(window, framebuffer_size_callback);
```

Осталась пара небольших деталей. Нам нужно написать цикл рендеринга (render loop), внутри которого и будет происходить вся магия. Без этого цикла наше окно тут же закроется. На каждой итерации происходит проверка - было ли сказану GLFW, чтобы он закрылся. Если да - выходим из цикла. Внутри же самого цикла проверяется, были ли какие-то события или нет (ввод с клавиатуры, мышь, менялся ли размер окна). Также происходит замена буффера. В OpenGL используется двойной буффер для того, чтобы избежать мерациния и подёргивания изображения при отрисовке. Это происходит потому, что отрисовка происходит в реальном времени. Чтобы этого избежать - используются два буфера. Передний буфер хранит в себе конечное изображение, пока всё остальное происходит в заднем. Как только все действия будут выполнены - мы поменяем их местами. 
``` c++
// render loop
while (!glfwWindowShouldClose(window)) {
    glfwSwapBuffers(window);
    glfwPollEvents();
}
```

Остался последний шаг - после выхода из цикла нужно очистить всё, что осталось от GLFW. 
``` c++
// закрыть glfw
glfwTerminate();
```
